# 游戏式 AI 角色世界 - 系统架构

## 核心理念

**这不是聊天机器人，这是一个世界模拟器。**

通用 agent 平台的目标是"完成任务"，而你的目标是"维持世界的可信运行"。这意味着：
- 角色可以拒绝用户（人格一致性 > 用户满意度）
- 事情会遗忘（记忆有损耗）
- 关系会恶化（不是永远正向）
- 时间会流逝（世界不等待用户）

## 核心模块设计

### 1. 人格核心 (Persona Core)

**问题：** 大多数 AI 角色在长对话中人格漂移

**设计：**
```
Persona = {
  identity: {          // 不可变的核心身份
    name, age, background, role_in_world
  },
  traits: {            // 稳定的人格特质（带权重）
    honesty: 0.8,      // 0-1 之间的倾向
    warmth: 0.6,
    curiosity: 0.7,
    ...
  },
  values: {            // 价值判断优先级
    loyalty > truth > comfort
  },
  speech_pattern: {    // 语言风格模板
    vocabulary_level, sentence_structure, catchphrases
  },
  boundaries: {        // 绝对不做的事
    "won't discuss: politics",
    "won't pretend to be human",
    ...
  }
}
```

**关键机制：**
- 每次生成回复前，用 Persona 作为 system prompt 的硬约束
- 定期自检：「这个回复符合我的人格吗？」
- 允许人格有「成长」，但必须是渐进的、有因果的

### 2. 记忆系统 (Memory System)

**问题：** 通用 RAG 只是检索，不是记忆

**设计：三层记忆模型**

```
┌──────────────────────────────────────────────┐
│              工作记忆 (Working)               │
│  当前对话的短期上下文，session 内有效           │
│  容量有限，会自然遗忘                          │
├──────────────────────────────────────────────┤
│              情景记忆 (Episodic)              │
│  具体事件的记录：「2026-03-09 和用户第一次见面」│
│  带时间戳、情绪标记、重要性评分                 │
│  可检索，但检索结果带「记忆模糊度」             │
├──────────────────────────────────────────────┤
│              语义记忆 (Semantic)              │
│  提炼的知识：「用户喜欢咖啡」「用户在上海」     │
│  从情景记忆中归纳，更稳定                       │
│  用于长期理解用户                              │
└──────────────────────────────────────────────┘
```

**记忆操作：**
- **编码：** 对话 → 提取关键事件 → 存入情景记忆
- **巩固：** 定期回顾情景记忆 → 提炼语义记忆
- **检索：** 根据当前上下文 + 关系状态 → 检索相关记忆
- **遗忘：** 低重要性记忆随时间衰减，可被「遗忘」

**关键技术点：**
- 记忆不是完美的，允许「记不清」「记错了」
- 记忆检索要考虑情绪状态（高兴时更容易想起开心的事）
- 记忆是角色私有的，不同角色对同一事件记忆不同

### 3. 关系系统 (Relationship System)

**问题：** 大多数 AI 对所有人一样好

**设计：**
```
Relationship = {
  character_id,
  user_id,
  metrics: {
    trust: 0.5,       // 0-1，基于历史互动
    intimacy: 0.3,    // 情感亲密度
    respect: 0.7,     // 尊重程度
    tension: 0.1      // 紧张/冲突程度
  },
  history: [          // 关键关系事件
    { type: "betrayal", date, impact: -0.3 },
    { type: "support", date, impact: +0.2 },
    ...
  ],
  status: "acquaintance" | "friend" | "close" | "strained" | "broken",
  unlocks: [...]      // 关系达到一定程度解锁的互动/剧情
}
```

**关键机制：**
- 关系变化是缓慢的，单次互动影响有限
- 关系状态影响角色行为（信任低 → 更保守）
- 关系可以恶化，不是单向上升
- 某些剧情/对话只在特定关系状态下解锁

### 4. 世界状态 (World State)

**问题：** 聊天机器人没有「世界」，只有对话

**设计：**
```
WorldState = {
  time: {
    current: ISO8601,
    scale: "real_time" | "accelerated" | "paused",
    events: [...]     // 计划中的世界事件
  },
  locations: {        // 世界中的场景
    id, name, description,
    characters_present: [],
    interactable_objects: []
  },
  active_events: [    // 正在发生的事件
    { id, description, started_at, expected_end, participants }
  ],
  character_states: { // 每个角色的当前状态
    character_id: {
      location,
      activity,
      mood,
      availability: "available" | "busy" | "unavailable"
    }
  }
}
```

**关键机制：**
- 世界在用户不在线时也在运行（角色有自己的生活）
- 角色可以「忙碌」，不是随时待命
- 事件有因果链，不是随机发生
- 用户可以错过事件（遗憾设计）

---

### 5. 交互层 (Interaction Layer)

**游戏式结构设计：**

```
┌─────────────────────────────────────────────┐
│              对话模式                        │
│  自由对话，但有角色人格约束                   │
├─────────────────────────────────────────────┤
│              行动模式                        │
│  用户可以选择「做什么」而不仅是「说什么」       │
│  例：[拜访角色] [送礼] [邀请外出] [询问往事]   │
├─────────────────────────────────────────────┤
│              事件模式                        │
│  世界事件触发，用户参与或旁观                  │
│  例：「角色生日」「节日活动」「突发事件」       │
├─────────────────────────────────────────────┤
│              成长模式                        │
│  关系/角色/世界的状态变化可视化                │
│  例：关系等级提升、解锁新对话选项              │
└─────────────────────────────────────────────┘
```

## 技术栈建议

### 核心服务
- **LLM 编排：** LangGraph / AutoGen（用于多角色交互）
- **记忆存储：** 
  - 向量数据库（Chroma / Qdrant）用于语义检索
  - PostgreSQL 用于结构化数据（关系、世界状态）
- **事件系统：** Redis Streams / Kafka（世界事件推送）
- **时间推进：** Cron + 状态机（处理离线时间流逝）

### 关键挑战与解法

| 挑战 | 解法 |
|------|------|
| 人格漂移 | 每次生成前注入 Persona 约束 + 定期自检 |
| 记忆过载 | 三层记忆 + 遗忘机制 + 重要性评分 |
| 关系虚假 | 变化速率限制 + 负面事件影响 + 不可逆伤害 |
| 世界空洞 | 预设事件库 + 程序化生成 + 角色自主行为 |
| 用户错过 | 事后告知 + 不可逆后果 + 「遗憾」作为设计元素 |

---

## 与通用 Agent 平台的区别

| 维度 | 通用 Agent 平台 | 游戏式角色世界 |
|------|----------------|---------------|
| 目标 | 完成任务 | 维持世界可信运行 |
| 记忆 | 任务上下文 | 角色人生经历 |
| 人格 | 可切换/可调整 | 稳定、有边界 |
| 关系 | 无/浅层 | 核心系统，会演化 |
| 时间 | 用户驱动 | 独立流逝 |
| 错误 | 需要修正 | 可以是剧情（遗憾） |
| 成功标准 | 任务完成度 | 情感投入度、世界沉浸感 |

## 下一步建议

1. **先做最小可行世界：** 1 个角色 + 基础记忆 + 简单关系
2. **验证核心循环：** 对话 → 记忆 → 关系变化 → 解锁新内容
3. **再扩展：** 多角色、世界事件、更复杂的互动
