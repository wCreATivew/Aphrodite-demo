# Demo 架构 v2 - 纯本地游戏内嵌方案

**核心原则：** 100% 本地运行，无远程服务器依赖，可选云端增强。

---

## 🎯 Demo 规格

| 项目 | 规格 |
|------|------|
| **角色数量** | 3 个主要角色 (Tier 1) + 10 个次要角色 (Tier 2) |
| **使用场景** | 游戏内嵌，单人使用 |
| **交互方式** | 文字对话 + 语音输出（可选语音输入） |
| **延迟要求** | 对话<1s，动作<2s |
| **硬件配置** | AMD 9950X + 32GB RAM + RTX 5090 (24GB) |
| **服务器** | 无远程服务器（纯本地） |

## 📊 性能预算分配

### 延迟分解（对话<1s）

| 步骤 | 耗时 | 优化方案 |
|------|------|---------|
| **1. 用户输入 → 记忆检索** | <50ms | FAISS 向量检索 |
| **2. 人格决策** | <10ms | 规则引擎 |
| **3. LLM 推理** | <600ms | 本地 7B 模型 + vLLM |
| **4. TTS 合成** | <300ms | GPU 加速 + 缓存 |
| **5. 游戏动画同步** | <50ms | 异步加载 |
| **总计** | **<1010ms** | 目标<1s |

### 显存分配（RTX 5090 24GB）

| 组件 | 显存 | 说明 |
|------|------|------|
| **LLM (Qwen2.5-14B)** | 12GB | INT4 量化 |
| **TTS (Bert-VITS2)** | 4GB | 3 个主要角色模型 |
| **TTS (GPT-SoVITS)** | 2GB | 10 个次要角色共享 |
| **FAISS 索引** | 2GB | 记忆向量索引 |
| **缓存/预留** | 4GB | 语音缓存 + 预留 |
| **总计** | **24GB** | 刚好够用 |

### 内存分配（32GB）

| 组件 | 内存 | 说明 |
|------|------|------|
| **游戏客户端** | 8GB | Unity/UE |
| **OpenClaw 运行时** | 4GB | 核心逻辑 |
| **LLM 推理** | 8GB | CPU 卸载 |
| **记忆系统** | 4GB | SQLite + 缓存 |
| **系统预留** | 8GB | 操作系统 + 其他 |
| **总计** | **32GB** | 合理分配 |

## 🎭 角色分级方案

### Tier 1: 主要角色 (3 个)

**代表：** 丰川祥子、高松灯、千早爱音

**配置：**
- **TTS:** Bert-VITS2 独立模型（高质量）
- **人格:** 复杂规则（200+ 行配置）
- **记忆:** 独立 SQLite 数据库
- **LLM:** 优先使用云端（质量优先）
- **显存:** 4GB/角色

**延迟目标：**
- 对话：<800ms
- TTS: <300ms

---

### Tier 2: 次要角色 (10 个)

**代表：** 长崎素世、椎名立希、NPC 等

**配置：**
- **TTS:** GPT-SoVITS 共享模型（中等质量）
- **人格:** 简化规则（50+ 行配置）
- **记忆:** 共享 SQLite 数据库
- **LLM:** 本地 7B 模型（速度优先）
- **显存:** 0.2GB/角色（共享）

**延迟目标：**
- 对话：<1s
- TTS: <150ms

## 🔧 技术选型

### LLM 推理

| 方案 | 模型 | 延迟 | 质量 | 推荐场景 |
|------|------|------|------|---------|
| **本地优先** | Qwen2.5-7B-INT4 | <400ms | ⭐⭐⭐⭐ | 日常对话 |
| **本地高质量** | Qwen2.5-14B-INT4 | <600ms | ⭐⭐⭐⭐⭐ | 复杂对话 |
| **云端 (可选)** | Qwen3.5-Plus API | 1-2s | ⭐⭐⭐⭐⭐ | 关键剧情 |

**推荐配置：**
```python
# 智能路由
if is_critical_plot():
    return cloud_llm(query)  # 云端，质量好
elif len(query) > 100:
    return local_14b(query)  # 本地 14B
else:
    return local_7b(query)   # 本地 7B，速度快
```

---

### TTS 合成

| 方案 | 模型 | 延迟 | 质量 | 显存 | 推荐场景 |
|------|------|------|------|------|---------|
| **Tier 1** | Bert-VITS2 | <300ms | ⭐⭐⭐⭐⭐ | 4GB | 主要角色 |
| **Tier 2** | GPT-SoVITS | <150ms | ⭐⭐⭐⭐ | 2GB(共享) | 次要角色 |
| **缓存** | 预合成音频 | <10ms | ⭐⭐⭐⭐⭐ | 0GB | 常用语句 |

**推荐配置：**
```python
# TTS 缓存策略
common_phrases = ["你好", "谢谢", "再见", "我明白了", ...]
# 预合成，直接播放音频文件

# 非常用语句
if text in cache:
    play(cache[text])
else:
    audio = tts.synthesize(text, emotion)
    cache[text] = audio
    play(audio)
```

### 记忆系统

**技术栈：**
- SQLite (文本存储)
- FAISS (向量检索)
- sentence-transformers (嵌入，BAAI/bge-small-zh)

**性能：**
- 检索：<50ms (FAISS)
- 写入：<20ms
- 索引大小：~2GB (10 万条记忆)

**配置：**
```python
# 每个角色独立记忆库
memory_stores = {
    "sakiko": MemoryStore("memory/sakiko.sqlite"),
    "tomori": MemoryStore("memory/tomori.sqlite"),
    "anon": MemoryStore("memory/anon.sqlite"),
    "npc_shared": MemoryStore("memory/npc.sqlite"),  # 10 个次要角色共享
}
```

## 🚀 启动流程

```
1. 游戏启动
   ↓
2. 加载 OpenClaw 运行时 (<5s)
   ↓
3. 加载 LLM 模型到 GPU (<10s, 首次启动需下载)
   ↓
4. 加载 TTS 模型到 GPU (<5s)
   ↓
5. 加载记忆系统 (<2s)
   ↓
6. 进入游戏 (<1s)
   ↓
✅ 总启动时间：<30s (首次启动额外 +20s 下载)
```

## ⚠️ 风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| **显存不足** | TTS/LLM 崩溃 | 动态卸载，降低质量 |
| **内存不足** | 游戏卡顿 | 限制记忆数量，定期清理 |
| **LLM 太慢** | 对话延迟>1s | 降级到 7B 模型 |
| **TTS 排队** | 语音延迟 | 预合成 + 队列管理 |
| **存档损坏** | 进度丢失 | 自动备份 + 多存档位 |
