## 📖 核心概念速览

### 情感三要素

```
情感 = 类型 + 强度 + 衰减

类型：开心/悲伤/愤怒/惊讶/恐惧/厌恶/中性/喜爱/兴奋/焦虑
强度：0.0 (无) - 1.0 (极强)
衰减：每秒减少的强度 (如 0.02 = 每秒减少 2%)
```

### 效价 - 唤醒度

```
         激动 (高唤醒度)
            ↑
            │
  愤怒 😠   │   兴奋 🤩
  恐惧 😨   │   开心 😊
            │
────────────┼──────────── → 正面 (高效价)
  负面      │      正面
            │
  悲伤 😢   │   (无)
            │
            ↓
         平静 (低唤醒度)
```

### 情感叠加规则

- **兼容情感** 可以共存 (如 开心 + 兴奋)
- **冲突情感** 会相互抑制 (如 愤怒 + 喜爱)
- **主导情感** = 强度最高的情感

## 🔬 实验场景推荐

### 实验 1: 观察情感衰减

1. 点击"赞美" → 开心达到 40%
2. 停止操作，观察 10 秒
3. 看开心情感从 40% → 30% → 20% → ... → 消失

### 实验 2: 情感叠加效果

1. 点击"赞美" → 开心 😊
2. 点击"悲伤消息" → 开心 + 悲伤共存
3. 观察主导情感在两者间切换

### 实验 3: 效价 - 唤醒度轨迹

1. 打开网页演示的状态图
2. 依次触发：赞美 → 悲伤 → 愤怒 → 惊喜
3. 看点在二维空间中的移动轨迹

### 实验 4: 调节衰减率

1. 将"衰减速率"滑块调到最大
2. 触发"赞美"
3. 看情感快速消失
4. 将滑块调到最小
5. 再次触发"赞美"
6. 看情感持久存在

## ❓ 常见问题

**Q: 情感为什么会消失？**
A: 情感会随时间衰减，当强度低于 5% 时视为消失。

**Q: 最多可以同时有多少种情感？**
A: 默认最多 5 种，可在配置中调整。

**Q: 如何添加新情感类型？**
A: 在配置文件的 `emotions` 部分添加新条目，定义其衰减率、效价等属性。

**Q: 可以连接 Live2D 或 VRM 吗？**
A: 可以！实现 `ExpressionAdapter` 接口即可。参考协议文档第 4 节。
