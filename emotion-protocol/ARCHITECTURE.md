## 2. 数据流图

### 2.1 事件触发流程

```
用户操作/外部事件
       │
       ▼
┌──────────────┐
│ 创建事件对象  │ TriggerEvent { id, type, payload, timestamp }
└──────────────┘
       │
       ▼
┌──────────────┐
│ 事件入队     │ 添加到事件队列 (支持优先级)
└──────────────┘
       │
       ▼
┌──────────────┐
│ 规则匹配     │ 遍历规则列表，查找匹配的条件
└──────────────┘
       │
       ▼
┌──────────────┐
│ 冷却检查     │ 检查目标情感是否在冷却中
└──────────────┘
       │
       ▼
┌──────────────┐
│ 情感修改     │ 创建新情感或更新现有情感强度
└──────────────┘
       │
       ▼
┌──────────────┐
│ 冲突解决     │ 处理情感叠加/冲突
└──────────────┘
       │
       ▼
┌──────────────┐
│ 状态更新     │ 重新计算主导情感、效价、唤醒度
└──────────────┘
       │
       ▼
┌──────────────┐
│ 通知表现层   │ 通过回调/WebSocket 推送状态变化
└──────────────┘
```

### 2.2 情感衰减流程

```
┌──────────────┐
│ 定时器启动   │ 每 100ms 触发一次
└──────────────┘
       │
       ▼
┌──────────────┐
│ 遍历所有情感 │ 获取活跃情感列表
└──────────────┘
       │
       ▼
┌──────────────┐
│ 计算衰减     │ intensity = initial * e^(-decayRate * elapsed)
└──────────────┘
       │
       ▼
┌──────────────┐
│ 检查消失     │ intensity < 0.05 → 移除情感
└──────────────┘
       │
       ▼
┌──────────────┐
│ 更新主导情感 │ 重新确定主导情感
└──────────────┘
       │
       ▼
┌──────────────┐
│ 通知表现层   │ 推送更新后的状态
└──────────────┘
```

## 8. 性能考虑

### 8.1 时间复杂度

| 操作 | 复杂度 | 说明 |
|------|--------|------|
| 触发事件 | O(n) | n = 规则数量 |
| 获取状态 | O(1) | 缓存计算结果 |
| 衰减计算 | O(m) | m = 活跃情感数量 |
| 冲突解决 | O(m²) | 情感对比较 |

### 8.2 优化建议

1. **规则索引**: 按事件类型建立规则索引
2. **惰性计算**: 仅在需要时计算效价/唤醒度
3. **批量更新**: 合并短时间内的多个事件
4. **节流推送**: 限制向表现层的推送频率

## 9. 线程安全

```
┌─────────────────────────────────────────────────────────────┐
│                      单线程模型 (推荐)                        │
├─────────────────────────────────────────────────────────────┤
│  - 事件处理在主线程                                         │
│  - 衰减计算在定时器回调                                     │
│  - 状态读取在渲染帧                                         │
│  - 无需锁，天然线程安全                                     │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                      多线程模型 (可选)                        │
├─────────────────────────────────────────────────────────────┤
│  - 使用读写锁保护状态                                       │
│  - 事件队列线程安全                                         │
│  - 衰减计算在独立线程                                       │
│  - 注意死锁风险                                             │
└─────────────────────────────────────────────────────────────┘
```
